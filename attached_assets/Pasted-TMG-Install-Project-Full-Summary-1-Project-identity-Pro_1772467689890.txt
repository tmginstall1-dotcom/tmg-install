TMG Install Project — Full Summary
1) Project identity

Project name: tmginstall project 1
Business: TMG Install / The Moving Guy Pte Ltd
Core service model:

Furniture installation

Furniture dismantling

Furniture relocation / moving

Mostly office + residential jobs

Main goal:
Build a real-world, production-safe AI-assisted furniture quoting and operations platform that allows customers to submit job requests, receive an estimated quote, pay a deposit, choose a booking slot, and then lets admin + staff manage the full job lifecycle.

3) Core business workflow (confirmed)

This is the workflow we have already defined as the correct one.

Step 1: Customer submits quote request

Customer fills quote form

Provides service type, address, item details, photos, and logistics info

AI parses the request and estimates pricing

Quote is created in database

Admin is notified of new quote request

Step 2: Admin reviews quote

AI quote is never fully final by default

Admin reviews item detection, breakdown, and totals

Admin approves / adjusts where needed

Step 3: Deposit request

After review, admin sends deposit payment request

Customer receives payment link (Stripe)

Step 4: Deposit paid

Stripe webhook confirms payment

Quote/job status updates automatically

Step 5: Booking slot selection

Booking is only confirmed after deposit is paid

Customer receives a secure booking link

Customer chooses a date/time from allowed slots

Admin can block dates/times

Rescheduling follows business rules

Step 6: Staff job access

Staff can only see the job after deposit payment

Staff receives job details, contact options, and service info

Step 7: Job execution

Staff can mark:

arrived

in progress

completed

Arrival/completion requires photo + GPS evidence

Step 8: Final payment (if needed)

Final invoice / payment link sent when applicable

Customer pays balance

Job is marked closed

4) Critical business rules already defined

These are important because they shape both frontend and backend logic.

Quote rules

AI quote should be as accurate as possible

Admin still reviews before money collection

Quote should show itemised breakdown

Quote should include:

item lines

quantities

service type per item

subtotals

discounts if applied

transport/logistics fees when applicable

GST is not included in the customer estimate flow unless intentionally added later

Service rules

Install + dismantle only may not require transport pricing

Relocation requires logistics calculations

Distance/floor/lift/access complexity matter for relocation pricing

Office and residential furniture must both be supported

Booking rules

Schedule is only confirmed after deposit paid

Customer self-books through secure link

Reschedule is self-serve only under rules:

future-only

not in-progress / not closed

at least 24h before job

1 free reschedule

within 24h → contact admin

Staff rules

Staff jobs visible only after deposit

Staff must be able to call / WhatsApp customer

GPS capture required on arrived/completed submissions

Photo evidence required on key milestones

My Recommended Upgrades

These are the upgrades I would add now to make your project much stronger and safer.

Upgrade 1: Lock the architecture baseline

Before adding more features, freeze a single approved structure.

What to lock

One final schema version

One final function name per action

One final status model

One final email trigger map

One final booking flow

Why

Right now, the biggest risk is not missing features.
It is version confusion and accidental duplicate logic.

My recommendation

Create a “TMG Stable Baseline” and treat everything else as patches only.

Example:

create-quote = only quote creation

admin-approve-quote = only approval

request-deposit = only deposit request

stripe-webhook = payment status only

send-booking-link = booking invite only

confirm-booking = booking confirmation only

request-final-payment = final payment only

No duplicate “v1/v2” unless absolutely necessary.

This makes debugging much easier:

You’ll know exactly where the issue is.
